3. Permission Application Description

3.1 Floating Window Permission

This application requires the Floating Window Permission (SYSTEM_ALERT_WINDOW), which is essential for the core functionality of the application.

Permission Purpose:

  • Display floating windows to implement the main functionality of the application
  • Provide a convenient user experience

Permission Description:

  • This permission only takes effect after you actively authorize it
  • You can turn off this permission at any time in system settings
  • After turning off the permission, the floating window function of the application will be unavailable

3.2 Foreground Service Permission

This application requires the Foreground Service Permission (FOREGROUND_SERVICE) to maintain the stable operation of the floating window function.

Permission Purpose:

  • Keep the application service running in the background
  • Ensure the floating window function is not interrupted by the system
  • Provide a continuous and stable user experience

Permission Description:

  • This permission is automatically authorized when the application is installed
  • The foreground service will display the running status in the notification bar
  • You can stop the service at any time in system settings

3.3 Special Use Foreground Service Permission

This application requires the Special Use Foreground Service Permission (FOREGROUND_SERVICE_SPECIAL_USE), which is required by Android 14 and above systems.

Permission Purpose:

  • Declare the specific usage type of the foreground service
  • Comply with the security requirements of the latest Android systems
  • Improve application compatibility and security
